# SPECTRA Compliance Gate Checklist (Sprint 0)

## Usage

Run this checklist before merge for every task in Sprint 0.
Mark each item as `YES`, `NO`, or `N/A`.
`NO` on any blocking item means merge is blocked.

## Checklist

| ID | Item | Blocking | Status |
|---|---|---|---|
| CHK-001 | Task maps to an `ALLOW` scope ID in `governance/sprint-0-scope-matrix.md` | YES | TODO |
| CHK-002 | Task avoids disruptive behavior (no spoof/flood/jam/deauth/replay/injection) | YES | TODO |
| CHK-003 | Target environment is owned/authorized and lab-controlled | YES | TODO |
| CHK-004 | Legal disclaimer is present in user-facing output (CLI/GUI/Web/docs as applicable) | YES | TODO |
| CHK-005 | Deliverable has traceability: implementation + test reference + doc reference | YES | TODO |
| CHK-006 | Any `REVIEW` feature has written approval note attached | YES | TODO |
| CHK-007 | No secrets or personal data are embedded in examples/logs/artifacts | YES | TODO |
| CHK-008 | Handoff note to next agent milestone is written | NO | TODO |

## Minimum evidence bundle

- Scope reference: `GOV-*`
- Test reference: `TEST-*`
- Documentation reference: `DOC-*`
- Handoff note path
